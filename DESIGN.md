# 设计文档 — AI-Voice-Drawing

> 版本:v0.1(开发开始前)| 本文档随开发持续更新,每个 PR 合并后同步勾选实现状态。

## 1. 产品定位

一款将自然语言编译为可编辑矢量场景的 AI 语音共创绘画工具。面向不会写 Prompt、缺乏绘画基础的用户:说出想法,AI 拆解为结构化图元逐步绘制;画面中每个元素都是独立对象,支持对象级语音修改、撤销重做与风格切换。

## 2. 核心数据流

```
麦克风 → Web Speech API → 文字实时上屏
      → LLM(场景编译器)→ JSON 命令序列
      → 命令执行器(逐条执行 + 绘制动画 + 快照入栈)
      → SVG 画布(可插拔风格渲染器)
      ↘ 元素树面板 / 版本历史(同一场景图驱动)
```

场景图(Scene Graph)为唯一数据源:画布渲染、元素树、撤销栈、版本历史全部由它派生。

## 4. 容错设计

- 语音识别中间结果实时上屏,用户始终知道系统"听到了什么"
- 解析结果以可读形式展示("理解为:绘制 1 个红色圆形")
- 无法解析 → 明确提示并建议换种说法,不静默失败
- 修改类指令执行后,被改对象闪烁高亮 + 文字反馈,配合撤销实现"先执行后反悔"
- 指代不明("把它变大"但无选中对象)→ 语音追问指哪个对象

## 6. 取舍说明

- 不引入扩散模型文生图:位图无法做对象级编辑,会破坏"可编辑矢量场景"的核心主线
- 不做 RAG:与核心创新无关,三天周期内优先保证编辑链路的完成度
- P0 必须全部完成;P2 视进度砍,砍掉的指令在第 3 节写明原因
- 视觉风格打磨放最后,但交互反馈(上屏、步骤高亮、状态提示)属于 P0 不延后
